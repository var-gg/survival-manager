# 기여 안내

이 문서는 `survival-manager` 저장소에 변경을 추가할 때 지켜야 할 기본 협업 규칙만 다룬다.

## 기본 원칙

- 작은 변경을 선호한다.
- 한 커밋은 한 목적만 담는다.
- 구현 전에 문서와 구조를 먼저 고정한다.
- 자동 생성 파일과 수동 작성 파일을 구분한다.

## 작업 시작 전

1. 현재 브랜치와 원격 상태를 확인한다.
2. 변경 목적을 한 문장으로 정리한다.
3. 이미 같은 목적의 문서가 있는지 확인한다.

## 커밋 규칙

- 커밋 메시지는 목적이 드러나게 쓴다.
- 구조 변경, 문서 변경, 구현 변경을 가능하면 분리한다.
- 초기 스캐폴딩성 변경은 `chore:`를 우선 고려한다.

## 문서 작성 규칙

- 문서는 한글로 작성한다.
- 코드, API, 파일명, 식별자는 영어를 유지할 수 있다.
- 한 문서에는 하나의 목적만 둔다.
- 의사결정은 문서로 남기고 채팅에만 의존하지 않는다.

## Unity 관련 규칙

- Unity 프로젝트 루트는 레포 루트다.
- `Assets/`, `Packages/`, `ProjectSettings/`는 루트 바로 아래 유지한다.
- 대용량 바이너리 도입 전 `.gitattributes`와 추적 정책을 먼저 검토한다.

## 검토 포인트

- 불필요한 생성물이나 캐시 파일이 포함되지 않았는가
- 문서 목적이 분리되어 있는가
- 루트 구조를 깨지 않았는가
- 후속 작업자가 이해할 수 있는가
