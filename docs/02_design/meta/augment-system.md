# Augment System

- Status: draft
- Last Updated: 2026-03-29
- Phase: prototype

## Purpose

This document defines the MVP augment structure.
The system should create run-level and light long-term differentiation without overpowering the base roster game.

## MVP Rules

### Temporary Augments

The MVP includes 9 temporary augments total:

- silver: 3
- gold: 3
- platinum: 3

Temporary augments are intended to shape one expedition or short-term run context.

### Combat Reward Delivery

Battle rewards may include a 3-choice temporary augment offer.
This should be one of the primary reward formats alongside gold and equipment.

### Permanent Augment Rule

The MVP includes 1 permanent augment slot.
Permanent augment acquisition should primarily come from expedition-end reward structure rather than routine combat nodes.

### Design Intent

Augments should:

- change how a squad behaves or scales
- create same-archetype variation through build direction
- interact with traits and equipment without requiring huge combinational depth
- remain understandable enough that a player can choose quickly

## Long-Term Expansion Points

- more augment tiers
- more permanent slots
- synergy-specific augment paths
- archetype-linked augment pools
- town or account-level progression layers

## Economic Runaway Risks

- if temporary augments stack too efficiently, they can erase base roster balance
- if permanent augments are granted too often, long-term power inflation will flatten decision quality
- if augment offers are too frequent, equipment and gold rewards lose value

## Balance Risks

- platinum augments may create runaway best-choice patterns
- permanent augment power may overshadow trait identity
- temporary augments may be too generic and fail to create meaningful build divergence

## Data Expansion Points

- temporary augment definitions
- permanent augment definitions
- reward-offer tables
- augment tags for synergy and class/race interaction

## Open Questions

- how often should temporary augment 3-choice rewards appear in a 5-node expedition?
- should permanent augments be chosen, earned randomly, or tied to milestone completion?
- how strong can temporary augments be without invalidating equipment and traits?
- what presentation is needed to make augment decisions legible in a fast prototype?
