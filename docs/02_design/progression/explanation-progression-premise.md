# Progression premise

- 상태: draft
- 최종수정일: 2026-03-31
- 단계: concept framing

## 목적

이 문서는 `survival-manager`의 현재 progression premise 가설을 정의한다.
progression이 왜 존재해야 하는지, 그리고 프로젝트를 너무 이른 시스템 깊이로 고정하지 않으면서 replayability를 어떻게 지지해야 하는지에 초점을 둔다.

## premise 가설

progression은 return motivation, build experimentation, 누적된 authorship 감각을 강화해야 한다.
서로 분리된 reward system을 쌓아 올리는 방식이 되어서는 안 된다.

## 검토 중인 progression 층위

- in-run growth
- loot acquisition
- crafting input 또는 upgrade material
- run 사이에 남는 persistent unlock
- 반복 플레이로 열리는 더 넓은 build option

## 제약

- progression system은 하나의 shared player fantasy를 향해야 한다.
- 추가되는 각 층위는 mental overhead를 정당화해야 한다.
- 반복 성장은 다시 run에 들어가고 싶게 만들어야 하지만, 초기 run을 무의미하게 만들면 안 된다.
- crafting은 maintenance busywork가 아니라 의미 있는 선택을 만들 때만 남긴다.

## 참고 메모

reference genre는 compounding run, retained gain, build variation 같은 동기 구조를 찾기 위한 참고일 뿐이다.
feature completeness의 템플릿으로 쓰지 않는다.

## progression 리스크

- loot, crafting, meta progression이 서로 중복된 층위가 될 수 있다.
- persistent growth가 run tension을 너무 약하게 만들 수 있다.
- progression이 expressive choice가 아니라 mandatory grind처럼 느껴질 수 있다.
- reward complexity가 combat loop가 실제로 흡수할 수 있는 범위를 넘을 수 있다.

## 열린 질문

- 장기 유지의 감정적 중심축이 되는 progression 층위는 무엇인가?
- crafting은 전략을 더하는가, 아니면 inventory friction만 늘리는가?
- 실패한 run 뒤에 무엇이 남아야 하는가?
- run tension이 무너지지 않으려면 permanent power는 어느 정도까지 허용되는가?
