## 목적

이 문서는 `survival-manager` 저장소의 문서 운영 기준을 정의한다. AI 에이전트와 사람이 같은 기준으로 문서를 수정할 수 있어야 한다.

## 1. 문서 계층 원칙

1. 문서는 폴더 계층으로 역할을 먼저 구분한다.
2. 상위 폴더는 주제를, 개별 문서는 단일 목적을 담당한다.
3. 인덱스 문서는 탐색 역할만 하며 상세 정책을 중복 서술하지 않는다.
4. 결정은 가능한 한 `04_decisions/`의 ADR로 분리한다.
5. 작업 절차와 운영 규칙은 설계 문서와 섞지 않는다.

## 2. 문서 메타데이터 규칙

모든 지속 문서는 본문 상단에 아래 메타데이터를 둔다.

- 제목: 문서의 공식 이름
- 상태: `draft`, `active`, `deprecated` 중 하나
- 소유자: 문서 유지 책임 주체
- 최종수정일: `YYYY-MM-DD`
- 소스오브트루스: 해당 주제의 기준 문서 경로
- 관련문서: 직접 연결된 문서 목록

간단 문서나 폴더 인덱스는 필요 시 축약할 수 있지만, 정책/절차/결정 문서는 메타데이터를 생략하지 않는다.

## 3. 문서 상태 변경, 폐기, 대체 규칙

- `draft`: 검토 중이며 기준 문서로 확정되지 않은 상태
- `active`: 현재 적용되는 기준 문서 상태
- `deprecated`: 더 이상 기준이 아니지만 참조 가치가 남아 있는 상태

상태를 바꿀 때는 아래를 함께 수행한다.

1. 메타데이터의 상태와 최종수정일을 갱신한다.
2. 대체 문서가 있으면 관련문서에 후속 문서를 추가한다.
3. 인덱스 문서에서 활성 기준이 무엇인지 반영한다.
4. 삭제 대신 `deprecated`를 우선 사용한다.

문서를 완전히 제거해야 할 경우에는 제거 사유와 대체 위치를 ADR 또는 동등한 기록으로 남긴다.

## 4. 문서 수정 시 동시 갱신 규칙

문서를 수정할 때는 본문만 바꾸지 않는다. 아래 항목을 함께 점검한다.

1. 들어오는 링크와 나가는 링크
2. 해당 폴더의 `index.md`
3. 문서의 `관련문서`
4. 상위 문서 또는 기준 문서의 참조 목록

링크가 깨지거나 인덱스가 오래된 상태로 남는 변경은 완료로 보지 않는다.

## 5. ADR 작성 트리거

아래 중 하나에 해당하면 ADR 작성을 검토한다.

- 폴더 구조를 바꾼다.
- 문서 운영 규칙을 바꾼다.
- 구현 방향에 장기 영향을 주는 기술 결정을 내린다.
- 여러 문서에 반복 적용될 기준을 새로 만든다.
- 기존 표준을 폐기하거나 대체한다.

작은 오탈자 수정이나 표현 개선만으로는 ADR을 만들지 않는다.

## 6. 문서와 구현이 충돌할 때 우선순위 기준

1. 보안, 배포, 운영 위험이 있는 경우 구현 상태를 즉시 확인하고 충돌 사실을 기록한다.
2. 의도된 최신 결정이 ADR에 있으면 ADR을 우선 기준으로 본다.
3. 구현이 이미 변경되었고 문서가 뒤처졌다면, 문서를 즉시 갱신하거나 문서 상태를 `draft` 또는 `deprecated`로 조정한다.
4. 문서와 구현 중 어느 쪽이 기준인지 불명확하면 임의 해석하지 말고 결정 기록을 남긴다.

원칙은 "침묵한 문서보다 검증된 현재 기준"을 우선하되, 충돌을 발견한 즉시 추적 가능한 기록으로 남기는 것이다.
