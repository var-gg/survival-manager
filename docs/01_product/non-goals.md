# 비목표

- 상태: draft
- 최종수정일: 2026-03-31
- 단계: prototype

## 목적

이 문서는 MVP가 매력적이지만 불안정한 범위를 흡수하지 않도록, 명시적인 비목표를 기록한다.

## MVP 비목표

아래 항목은 MVP 범위 밖이다.

- PVP
- advanced crafting
- seasonal balancing tool
- live backend service
- large-scale asset integration

## 추가로 비목표로 보는 항목

slice가 나중에 정당화하지 않는 한, 아래도 deferred로 본다.

- deep live-service 스타일 progression 운영
- 넓은 콘텐츠 수량 목표
- high-end cinematic presentation
- 과도한 unit customization layer
- slice 필요를 넘는 production-grade content pipeline 확장

## 왜 비목표인가

MVP는 아래의 좁은 루프를 증명해야 한다.

- roster pressure
- synergy value
- readable auto-battle
- expedition과 return 구조

이 증명을 강화하지 않는 기능은 뒤로 미룬다.

## 장기 비전 메모

현재 비목표 중 일부는 MVP 검증 이후 미래 후보가 될 수 있다.
하지만 그것이 현재 목표의 일부라는 뜻은 아니다.
MVP는 지금 무엇을 증명하는지로 평가해야지, 언젠가 무엇을 넣을 수 있는지로 평가하면 안 된다.

## 열린 질문

- MVP에서 basic crafting을 완전히 뺄 수 있는가, 아니면 가벼운 equipment flow는 필요해지는가?
- 이 프로젝트에서 "large-scale asset integration"은 실제로 어디부터인가?
- 지금 가장 유혹적이지만 정당화가 약한 deferred feature는 무엇인가?
