# release-checklist

## Purpose

Use this skill to structure release-readiness documents for the game, especially around Steam preparation.
This skill is documentation- and checklist-oriented, not automation-oriented.

## Applies To

- creating Steam release preparation checklists
- organizing launch-readiness documentation
- tracking release gates and missing items
- documenting release dependencies, assumptions, and review points
- clarifying what must be ready before store or build submission

## Does Not Apply To

- executing store publishing actions
- building deployment automation scripts
- changing gameplay systems unless directly required for checklist accuracy
- modifying original vendor contents under `Assets/ThirdParty`
- handling unrelated repository governance tasks

## Guardrails

- Keep the checklist practical and repository-relevant.
- Separate current state, required evidence, and pending risks clearly.
- Do not directly modify original vendor contents under `Assets/ThirdParty`.
- Do not silently widen scope into launch execution or automation.

## Expected Outputs

- release readiness docs
- Steam-oriented checklists
- gate criteria summaries
- documented open risks or blockers before release
